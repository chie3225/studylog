# スタディログ セットアップ手順

ログイン不要・リンクを知っている人なら誰でもアクセスできる形にするための手順です。
コマンド入力はほぼ不要で、すべてブラウザの画面操作でできます。

---

## 全体の流れ

1. Anthropic APIキーを取得する(AI採点・クイズ生成・課題表読み取り・分析用)
2. Supabaseでデータベースを作る(提出記録・予定・未提出理由の保存用)
3. GitHubにこのフォルダの中身をアップロードする
4. Vercelでそれを取り込んで公開する
5. Vercelに「環境変数」を設定する
6. 完成したURLを息子さん・先生に共有する

所要時間の目安: 30分〜1時間(初回のみ)

---

## 1. Anthropic APIキーを取得する

1. https://console.anthropic.com にアクセスし、アカウントを作成(claude.aiのアカウントとは別物です)
2. 支払い方法(クレジットカード)を登録する。従量課金です
3. 左メニューの「API Keys」→「Create Key」でキーを発行する
4. 表示された文字列(`sk-ant-...`から始まる)をどこかにメモしておく(二度と表示されません)

---

## 2. Supabaseでデータベースを作る

1. https://supabase.com にアクセスし、アカウントを作成
2. 「New Project」でプロジェクトを作成(名前は自由、パスワードは自動生成でOK。忘れずにメモ)
3. プロジェクトができたら、左メニューの「SQL Editor」を開く
4. このフォルダの `schema.sql` の中身を全部コピーして貼り付け、「Run」を押す(3つのテーブルが作られます: submissions, plans, missed_reasons)
5. 左メニューの「Project Settings」→「API」を開き、以下の2つをメモする
   - **Project URL**(`https://xxxxx.supabase.co` の形)
   - **service_role key**(「anon key」ではなく「service_role」の方。これは秘密情報なので取り扱い注意)

---

## 3. GitHubにアップロードする

1. https://github.com にアクセスし、アカウントを作成
2. 右上の「+」→「New repository」で新しいリポジトリを作成(例: `studylog`。Publicで構いません)
3. 作成後の画面で「uploading an existing file」というリンクをクリック
4. このフォルダの中身(`index.html`、`app.js`、`package.json`、`schema.sql`、`api`フォルダごと)をすべてドラッグ&ドロップでアップロードする
5. 「Commit changes」を押す

コマンドライン(git)は使わなくてOKです。ブラウザの操作だけで完結します。

---

## 4. Vercelで公開する

1. https://vercel.com にアクセスし、「Continue with GitHub」でアカウントを作成(GitHubと連携させます)
2. 「Add New...」→「Project」を選ぶ
3. 先ほど作ったGitHubリポジトリ(`studylog`)を選んで「Import」
4. 設定画面が出てきますが、特に何も変えずにこの時点ではまだ「Deploy」を押さないでください(先に環境変数を設定します。もし押してしまっても後から設定できます)

---

## 5. 環境変数を設定する

デプロイ設定画面(または、デプロイ後は「Settings」→「Environment Variables」)で、以下の3つを追加します。

| Key | Value |
|---|---|
| `ANTHROPIC_API_KEY` | 手順1でメモしたAnthropicのAPIキー |
| `SUPABASE_URL` | 手順2でメモしたProject URL |
| `SUPABASE_SERVICE_KEY` | 手順2でメモしたservice_role key |

3つとも入力したら「Deploy」(またはすでにデプロイ済みなら「Redeploy」)を押します。

---

## 6. 完成

デプロイが終わると、`https://studylog-xxxx.vercel.app` のようなURLが発行されます。

- このURLをそのまま息子さんのiPad・先生・さえさんのiPhoneに共有すればOK
- ログイン不要で、誰でも同じデータを見られます
- ホーム画面に追加すればアプリのように使えます

---

## 使い方の流れ

### 生徒側(「きょうの課題」タブ)
- 国語・数学・英語(書きまくり)・英語(予習)は毎日表示されます
- 理科・社会は、「管理」タブで課題表を同期した日にだけ表示されます
- 数学・理科・社会・国語(漢字ノート)は、**自分で丸つけをした後の写真**をアップロードしてください。AIは正誤を判定せず、写真に書かれた○×の跡だけを読み取ります
- 丸つけの跡が見つからない場合は「丸つけをしましょう」という警告が出ます
- ×がついた問題だけ、AIが解説と類似問題を出します。「とけた!」を押すと解決済みになります

### 管理側(「管理」タブ)
- **カレンダー**: 学校の課題表(スプレッドシート)のスクリーンショットを貼ると、AIが日付ごとの教科・課題を読み取って自動反映します。提出のないまま日が過ぎた日は、理由(体調不良/予定が詰まっていた/遊んでた)を記録できます
- **テスト前やり直し**: 過去90日分の、まだ解決していない間違いだけをまとめて確認・解き直しできます
- **分析**: 蓄積されたデータから、単元ごとの「つまずきポイント」「克服のコツ」をAIが文章でまとめます

---

## 困ったときは

- 生徒側で採点・クイズ生成が失敗する → Vercelの「Settings」→「Environment Variables」で`ANTHROPIC_API_KEY`が正しく入力されているか確認
- カレンダーや分析が空になる → `SUPABASE_URL`・`SUPABASE_SERVICE_KEY`が正しいか、`schema.sql`を実行し忘れていないか確認
- 課題表の同期が失敗する → スプレッドシートの共有設定は関係ありません(画像として直接AIに読ませているため)。画像が不鮮明な場合はうまく読み取れないことがあるので、明るい場所で撮り直してみてください
- 変更を反映したい → GitHub上でファイルを編集して保存すると、Vercelが自動的に再デプロイします

## 費用の目安

Anthropic APIの利用料は、1回の写真採点・クイズ生成あたり数円程度です。毎日6教科分を使っても、月あたり数百円〜千円程度に収まる見込みです(利用量によって変動するので、Anthropicのコンソールで時々確認することをおすすめします)。
